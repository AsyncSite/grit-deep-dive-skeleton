# 문제 정의

## 배경

커머스 서비스에서 한정 수량 상품을 판매합니다.

상품 `sku:limited`의 재고는 1개입니다. 이벤트 시작 시각에 100명이 동시에 결제를 시도합니다.

Redis lock 없이 단순히 `GET inventory -> if > 0 -> SET inventory - 1`로 처리하면 두 요청이 같은 재고를 읽고 둘 다 성공할 수 있습니다.

## 목표

Redis 기반 lock을 구현해서 동시에 여러 요청이 들어와도 재고 1개가 정확히 1번만 차감되게 만듭니다.

## 제약

- Redis 단일 인스턴스를 사용합니다.
- 회사 코드, 회사 데이터, 실제 결제 API를 사용하지 않습니다.
- 구현 언어는 Java 21, 프레임워크는 Spring Boot로 고정합니다.
- Redlock 구현은 필수가 아닙니다. 대신 왜 채택하지 않았는지 또는 언제 필요해지는지 설명해야 합니다.

## 금지

- `SETNX` 후 별도 `EXPIRE` 호출로 TTL을 붙이는 구현
- token 검증 없는 `DEL lock_key`
- `time.sleep()`으로 우연히 race를 피하는 구현
- 테스트만 통과시키는 hard-code

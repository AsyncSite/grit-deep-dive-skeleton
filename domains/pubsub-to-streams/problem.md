# 문제 정의

## 배경

Pub/Sub은 subscriber가 연결되어 있을 때만 메시지를 받습니다. consumer가 잠시 내려가 있으면 그 사이 메시지는 사라집니다.

주문 이벤트, 알림 재시도, 작업 큐처럼 재처리가 필요한 영역에서는 이 특성이 사고가 됩니다.

## 목표

Pub/Sub 방식의 유실 가능성을 Redis Streams consumer group 방식으로 바꿉니다.

## 제약

- Redis 단일 인스턴스를 사용합니다.
- Kafka로 옮기는 것은 이번 과제의 구현 범위가 아닙니다.
- 회사 코드, 회사 데이터, 실제 주문 이벤트를 사용하지 않습니다.
- 구현 언어는 Java 25, 프레임워크는 Spring Boot로 고정합니다.

## 금지

- Pub/Sub listener만 붙이고 유실 문제를 해결했다고 주장하는 구현
- 메시지 처리 후 ack를 생략하는 구현
- pending 상태를 관찰하지 않는 구현
- 테스트만 통과시키는 hard-code
